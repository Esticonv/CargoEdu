#Сценарии

##1.Запрос расчёта возможных вариантов доставки

Клиент предоставляет данные о грузе, места отправки и доставки.
В ответ получает 0...3 варианта с указанием продолжительности доставки (время отправления, время прибытия), стоймости

UI -> DC запрос вариантов расчётов (in: from, to, cargo.size)
DC -> MM запрос свободных машин
MM -> DC возврат доступных машин и себестоймости
DC -- Расчёт вариантов и сохранение результатов
DC -> UI возврат вариантов расчётов (out[]: calculationId, deliveryTime, cost )

##2.Подтверждение заказа

Повторный запрос через сервис расчёта для того чтобы клиент не мог подменить результаты

UI -> O Запрос на оформление заказа (in: id выбранного варианта)
O  -> DC запрос результата расчётов (in: id)
DC -> O  возврат (out: calculationId, deliveryTime, cost)
O  -- Оформление расчёта
O  -> UI возврат статуса что всё сделано
